# Antigravity 実装指示: ガントチャートの年/月/週モードの正式実装 + 休日グレー化

対象: `frontend/src/components/GanttChart.vue`（および必要なら `frontend/src/assets/frappe-gantt.css`）。
ライブラリ: **frappe-gantt 1.2.2**（`node_modules/frappe-gantt` にインストール済み。カスタム `view_modes` オブジェクト / `holidays` / `ignore` オプションに対応）。

## 背景（現状の問題）
前回の指示で「年/月/週の3モード切替」を依頼したが、現状は **frappe-gantt 既存モード（`Week`/`Day`/`Quarter Day`）にボタン名を付け替えただけ** で、仕様の詳細を満たしていない。また休日ハイライトが **ピンク**（仕様はグレー）。本指示で正式に作り直す。

## すでに実装済み（変更しないこと・壊さないこと）
- ガント枠の縦フル表示（`.gantt-wrapper { height: calc(100vh - 240px) }`）→ 維持。
- ステータス色分け（`bar-todo`/`bar-progress`/`bar-done`/`bar-delayed`）と凡例カード → 維持。遅延=赤文字、バーと文字が同系色にならない方針も維持。
- バードラッグでの日程保存（`on_date_change` → `store.updateTaskDates`）→ 維持。
- リソースマスタ / カレンダーマスタ（管理画面）/ 認証・ロール → 本件では触らない。
- 休日データは `store.calendarHolidays`（`[{date:"YYYY-MM-DD", ...}]`）で取得済み。これを月モードのグレー表示に使う。

## 実装タスク

### 1. 3モードを「カスタム view_modes オブジェクト」で正式実装する
ボタンの付け替えではなく、frappe-gantt 1.2.2 の **カスタム view_modes 定義**（`name` / `step` / `column_width` / `padding` / `lower_text` / `upper_text` / `thick_line` 等）を用いて、下記3モードを定義する。`Gantt` 初期化時の `view_modes` に独自モード3つを渡し、`view_mode` でデフォルトを指定、ボタンから `change_view_mode(<name>)` で切替える。
※フィールド名・シグネチャは `node_modules/frappe-gantt/dist/*.js` の実装（内蔵 DAY/WEEK/MONTH 定義）を必ず参照して合わせること。`lower_text`/`upper_text` は書式文字列または `(date, last_date, lang) => string` の関数で指定できる。

#### モード「年」（全体俯瞰・最小単位=週）
- 最小単位は **週**。各列＝1週間。週の起点は **月曜始まり**、列ラベルは**その週の月曜日の日付**（例: `1日`, `8日`）。
- ヘッダは階層表示:上段に **年（例: 2026年）と月（例: 6月）**、下段に **週（月曜日の日付）**。
- 3〜6か月のプロジェクト全体が無理なくおさまる列幅にし、1年超は横スクロールで見られる程度を目安にする。

#### モード「月」（当月網羅・最小単位=日）
- 最小単位は **日**。各列＝1日。
- ヘッダ:上段に月（例: 6月）、下段に日（例: 1, 2, 3…）。
- **非稼働日（土日 + `store.calendarHolidays` の祝祭日・企業休日）の列背景を「グレー」で表示**（現状のピンク/極薄ホワイトを廃し、視認できるグレー、例 `rgba(128,128,128,0.18)` に統一）。

#### モード「週」（1週間/1日詳細・最小単位=時）
- 最小単位は **時**。ただし24分割ではなく **6時間刻みの4分割**。時刻ラベルは **`00` / `06` / `12` / `18`**。
- ヘッダ:上段に日付（例: 6月10日）、下段に時刻（`00`,`06`,`12`,`18`）。

### 2. 休日グレー化（`holidays` オプションの修正）
- `holidays` オプションを、土日 + DB休日を **同一のグレー**で塗るよう修正する。関数形式が使えるため、例:
  ```js
  const isHoliday = (date) => {
    const day = date.getDay();              // 0=日,6=土
    const y = date.getFullYear();
    const m = String(date.getMonth()+1).padStart(2,'0');
    const d = String(date.getDate()).padStart(2,'0');
    const ds = `${y}-${m}-${d}`;
    return day === 0 || day === 6 || store.calendarHolidays.some(h => h.date === ds);
  };
  // 例: holidays: { 'rgba(128,128,128,0.18)': isHoliday }
  ```
- 色キーの指定方法・複数条件のまとめ方は frappe-gantt 1.2.2 実装に合わせる（色キー→関数 のマップが使える）。
- グレー表示は主に「月」モードで意味を持つ。年/週モードでの見え方が破綻しないことも確認する。

### 3. ボタンUIの整合
- 「時間粒度」ボタン群（`年 / 月 / 週`）の `:active` 判定と `changeViewMode` を、上記カスタムモード名（例: `'Year'`/`'Month'`/`'Week'` など定義した name）に合わせて修正する。既存の `'Week'`/`'Day'`/`'Quarter Day'` 直書きを置換。
- デフォルト表示モードは「月」を推奨（当月網羅が日常運用で使いやすいため）。必要なら変更可。

### 4. 再描画・整合
- モード切替・`store.tasks`/`planBGhostActive`/`calendarHolidays` 変更時の再描画（既存の watch + `initGantt`）でモードが保持されること。`change_view_mode` 後に再 `initGantt` してもユーザー選択モードが失われないようにする。
- ポートフォリオ用（`:is-portfolio="true"`）でも同じモード定義が機能すること。

## 受け入れ条件（実装後に dev サーバで実機確認し報告）
1. 「年」モード:列が**週単位**で、ラベルが**月曜始まりの日付**。上段に年・月が表示される。
2. 「月」モード:列が**日単位**で、**土日・祝日・企業休日の背景がグレー**。
3. 「週」モード:列が**6時間刻み**で、時刻ラベルが **00/06/12/18**。
4. 3モードのボタンで表示が**実際に切り替わり**、再描画後も選択が保持される。
5. 休日ハイライトが**グレー**（ピンクではない）。`AdminView` のカレンダーマスタで休日を追加すると、月モードの該当日がグレーになる。
6. ステータス色分け・凡例・遅延赤文字・バー/文字の同系色回避・全高レイアウト・ドラッグ保存が**従来どおり維持**されている。
7. コンソールに致命的エラーが出ない（Vuetify の既存 deprecation 警告以外）。

## 実装上の注意
- frappe-gantt 1.2.2 のカスタム view_mode / holidays / ignore の**正確なフィールド名と挙動は必ず `node_modules/frappe-gantt` の実装で確認**してから実装すること（バージョン差異で API 名が異なるため、推測で書かない）。
- 既存の機能（凡例・色・ドラッグ保存・全高・管理画面）には手を入れない。差分は GanttChart のモード定義と holidays 設定に限定する。
